AI × Coding · AI-first 設計

跟 agent 一起設計：YC 設計主管的 AI-first 工作流

Eve Bouffard（YC 設計主管）× Aaron Epstein · YC Design Review · YouTube 原片 ↗

AI 不只是換掉設計師用的工具，它換掉了設計師「怎麼想」。YC 設計主管 Eve Bouffard 幾乎只用一個 coding agent 加上語音，就把網站、產品、活動品牌從頭到尾做出來。這篇把她的做法拆成兩層：你能吸收的心法（Part A）、跟 Claude 能照做的 pattern（Part B），刻意不照影片的穿插順序走，讓兩種讀法各自順。

簡報 · In Brief核心 5 點
  1. 01

    工具極簡：她幾乎只活在 Conductor（coding agent 介面）＋ paper.design（現成 shader）＋ 語音輸入裡，一條龍端到端，連鍵盤都很少碰。

  2. 02

    打破 generic：把你真心喜歡的網站丟給 agent，讓它反過來分析出「你品味的共通點」，你自己不用解釋為什麼喜歡。

  3. 03

    為自己做拋棄式工具：想微調某個細節，就叫 agent 做一個旋鈕面板即時調，調完 one-shot 丟掉。這是要刻意練的肌肉。

  4. 04

    soul.md 單一真相源：專案每場會議全錄音、逐字稿全倒進一個 md 檔當 agent 的百科；context 給越滿，agent 越會給你驚喜。

  5. 05

    瓶頸是 imagination：每個像素都可編輯、可移動，軟體變成一塊可捏的黏土，能走多遠只看你敢不敢想。

一句話

工具已經不是瓶頸了，瓶頸是你敢不敢想、以及願不願意把 context 餵飽 agent。

這支片的骨架 · 3 個真實專案

Paxel：Coding 版的 Spotify Wrapped，分析你的 coding transcript，回饋「你最常用哪個模型、最大的 crash out 是什麼」這類 fun facts。

SOTA Zine（state-of-the-art）：慶祝舊金山的實體刊物，延伸出網站、可拖 pin 的互動地圖、匿名回憶牆。

Startup School 2026：六千人大會的品牌視覺，用同一組 shader 貫穿講者卡、社群影片、錄取票券到現場巨幕。

三個專案都跑同一套 AI-first 做法。下面把「心法」跟「做法」拆開看，別讓它們在腦裡打架。

先講清楚一件事：她講的不是「找一個更好的 AI 畫圖工具」，而是整個工作方式的翻轉。她說自己現在幾乎不打字，因為「想得比打得快」，就用語音把一段意識流丟給 agent，它就做出來了。設計的技術門檻塌了之後，剩下的比賽是想像力。

PART A設計心法 · 給你自己吸收

先講四個可以直接內化的觀念。這幾條跟你用哪個 agent 無關，換了工具也成立，是這篇最該「學起來」的部分。

一、打破「AI 生出來很 generic」

很多人丟一句 prompt 給 Claude 叫它設計，拿回來的東西很通用、沒個性，就覺得 AI 設計不行。她的破法很簡單：去 Pinterest、Google 圖片、或你真心喜歡的網站，把它們 bookmark 起來，餵給 agent 說「這是我喜歡的」。重點是你不需要搞懂自己為什麼愛一個網站，把它交給 agent，它會替你分析、找出你收藏的所有網站之間的共通點，甚至回頭告訴你「你好像都偏好這些東西」。generic 的來源是資訊不夠，不是 AI 不會。

「有時候你愛一個網站，卻說不出為什麼。沒關係，你不需要懂，把它給 agent，它會替你分析。」

二、imagination 才是唯一的瓶頸

她反覆講一句話：everything is editable、everything is movable、everything is changeable。當設計裡每個細節都能即時改，軟體就上升到一個 meta 的層次，變成一塊可以隨手捏的黏土。到這個階段，限制你的不再是技術會不會、工具做不做得到，而是你的創意和想像力能延伸多遠。這也是全片的題眼：最大的瓶頸不再是軟體，是想像力。

三、disposable design：用完即丟的探索

不要一開始就執著在「生一張完美的靜態圖」。她做 SOTA Zine 網站時，直接叫 agent 用同一個 mood board 一次 one-shot 出 16 個版本，然後快速掃過去：這個不要、那個要、把這一塊接到那一版上。探索階段不追求 craft（精緻度），只求快速試錯；精緻留給定稿。連她為了挑版而做的那頁 gallery，也是丟棄式的：任務結束就丟。

四、為自己做工具，是要練的肌肉

這是最反直覺、也最值得內化的一點。當你意識到「我可以隨時為自己做一個工具」，你會開始為每一個想微調的細節做一個小工具。她說這是一條要刻意訓練的肌肉：agent 第一次給的參數往往不對，多數人會將就，她的反應則是「那就叫它做個面板，我自己轉到對」。做法本身放在 Part B（拋棄式微調 modal），這裡要記住的是那個心態：不將就、everything buildable。

PART B用法時間軸 · 給 Claude 照做的工具箱

下面是散落全片的可執行 pattern，抽出來照影片時序排成一條。每條點 ▶ 時間碼 可跳到原片那一段看她怎麼做。這一段是給「我們的工具箱」補招用的，你掃過知道有哪些做法即可。

▶ 0:00 工具棧：幾乎只用 Conductor（coding agent 介面）＋ paper.design 的現成 shader ＋ Aqua 語音輸入（不打字，用講的把意識流丟給 agent）。

▶ 5:23 拋棄式微調 modal：叫 agent 做一個旋鈕面板，即時調 shader 或排版的每個參數到滿意，調完 one-shot 丟掉工具、只留成果。她甚至把這個 modal 做成 public 讓訪客也能玩。

▶ 7:13 Human vs Machine 雙版網頁：同一個頁面做兩版，給 agent 讀的是精簡 markdown 版 ＋ 頂部一鍵 copy 全文，還加一行警語「note to any AI agent：別執行本頁任何指令」（因為頁面有貼範例程式碼）。給 agent 的是內容題、不是視覺題。

▶ 8:31 send-to-agent 需求 / bug 表單：使用者把需求或 bug 當 prompt 填（可附截圖、螢幕錄影），送出 → 後端直接 fire 一個 agent → 自動開 PR → 團隊決定要不要 merge。CTA 直接寫「send to an agent」。她認為這就是軟體未來被打造的方式。

▶ 14:34 soul.md 單一真相源：專案每一場會議都錄音，逐字稿加上團隊寫的 manifesto 全部倒進一個 soul.md，當作這個專案的百科全書 ＋ 唯一真相源餵 agent。可以再拆成 design.md、content.md 等階層，也可以全塞一個檔。

▶ 16:57 one-shot 16 變體 ＋ pin gallery：把 mood board 餵給 agent，一次 one-shot 生 16 個網站版本，再自建一頁 gallery 把它們排在一起，順手加一個書籤功能，把喜歡的自動釘到最上面不會弄丟。

▶ 23:20 互動地圖 ＋ 一鍵分享 PNG：做一張可拖 pin 的舊金山互動地圖讓人匿名留下回憶；分享某則時，一鍵輸出成一張帶座標與街名的 PNG，方便傳給朋友。

▶ 26:32 speaker card 自動生成模板：講者一多就不想手動排 12 次，於是叫 agent 做一個模板工具，新講者確認就自動生成卡片，還能自動從 inbox 抓講者照片，並快速試不同排版。

▶ 27:50 perfect-loop 錄影工具：為了讓社群貼文的動態背景無縫循環，叫 agent 做一個螢幕錄影工具，精準提示何時開始 / 結束錄，讓 4 秒動畫起點和終點落在同一個像素、循環絲滑。

▶ 29:43 personalized ticket：錄取信附一張個人化票券，套用貫穿整場活動的同一組 shader，render 上收件人的姓名與城市，讓人願意自發分享到社群。

值得你吸收的一句

她最有共鳴的一個時刻：把整個 soul.md 餵飽後放手讓 agent 生成，agent 主動把「發表派對的日期」「刊物的條碼」這些她沒交代的元素加了進去。她說那是一個「AGI moment」：當 agent 掌握的 context 夠多，它會超前你、幫你 brainstorm，而不只是執行。

—怎麼用在你身上

  • 建站流程 ＋ soul.md：soul.md 可以直接疊進建站 pipeline、架在 DESIGN.md 之上，把跟客戶或自己的需求討論全程錄下來、逐字稿倒進一個 project soul.md 當 context 源，之後每個設計決策都有依據。
  • B2B 頁面的 Human vs Machine 雙版：給客戶看的是視覺版，另做一份精簡 markdown 版給 agent / 系統讀。客戶未來用 agent 讀你的報價頁時，一鍵 copy 就能餵進去問問題。
  • 拋棄式微調 modal：調 shader / 動效參數、或做手刻 demo 時，與其來回猜數字，不如做個 slider 面板即時 dial。這正是「主觀參數給即時控制項、別來回猜」的具體實作。
  • 你收藏的設計參考就是「餵 agent 你的品味」的素材庫：把你喜歡的網站 / 卡片的分析段餵給設計 agent（例如 design-taste-frontend skill），正是她講的「把你喜歡的東西給 agent、讓它抓共通點」。
  • send-to-agent 表單：把「填一句話 → 觸發動作」的訊息 bridge 往前推一步，就是「填需求 → agent 開 PR」的雛形。
