情感化設計：讓 App 有生命感

Tim Gabe · YouTube《The Secret Behind Weirdly Addictive Apps》· Duolingo / Phantom / Revolut 案例
簡報 · In Brief核心 5 點
  1. 01

    有用，已經不是護城河：功能已被拉平，長期差異化落在「打開 app 的感覺」。

  2. 02

    微互動即時回饋：重複行為每完成一次，別只給綠勾，給 bounce／glow／sparkle，讓人覺得被鼓勵、被回應。

  3. 03

    慶祝小勝 + 進度感：用動畫呈現 streak、等級、完成數，給「正在累積、正在建造」的感覺，強化回訪。

  4. 04

    拋光＝信任：金融／crypto／健康這類高風險領域，把 motion、transition、視覺細節當核心功能不是裝飾，每個微互動都是信任訊號。

  5. 05

    設計換算成數字：Duolingo、Phantom、Revolut 三個案例，設計改動都直接對應到 DAU、排名、營收的變化。

一句話

有用已經不夠，情感回饋迴路才是留住使用者的長期護城河，三個真實案例證明設計能直接換算成商業數字。

02
Type BAI × Coding · 餵我能力

MLX 本機模型服務：Mac 上跑 agentic AI

Apple WWDC26 Session 232 · MLX team · 不上雲、無 API key、零用量成本
能力 · Pattern四層拼法
  1. 01

    四層 stack：MLX（Apple Silicon 陣列框架）→ MLX-LM（載入／量化／微調）→ MLX-LM Server（OpenAI 相容 HTTP，支援 tool calling）→ Agent 層；Ollama／LM Studio／vLLM 都已建在 MLX 上。

  2. 02

    三步上線：pip install mlx-lm → mlx_lm.server --model ... → 把 agent 的 baseURL 指到 http://127.0.0.1:8080/v1，agent 不知道也不在乎模型在本機還是雲端。

  3. 03

    M5 加速：Neural Accelerators 讓矩陣乘法比 M4 快 4 倍，agentic loop 大多 token 是讀進來的 context，這塊 prompt processing 加速很有感。

  4. 04

    Continuous batching ＋分散式推論：多 subagent 同時打 server 時動態併批；模型太大塞不下單機時，用 mlx.launch + hostfile shard 到多台 Mac。

一句話

可複用的是「把 agent 指到本機 OpenAI 相容 endpoint」這招，跟現有的 GLM 路由是同一套機制的本機免費版，換來隱私與零成本，但能力遠不如 Opus、不取代現在的工作流。
