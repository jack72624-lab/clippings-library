Claude Tag：Slack 裡的常駐 AI 同事

AI Daily Brief（英文新聞）
簡報 · In Brief核心 5 點
  1. 01

    定義（官方）：Claude Tag 是「Claude Code 的進化，更主動、為整個團隊設計」。@ 一下，它把任務拆階段、用手上的工具做完、在 thread 回覆成果。

  2. 02

    一頻道一個共享 Claude：跟著頻道累積脈絡，隊友能接手你的進度；開 ambient 模式，它會主動跟進冷掉的討論串、跨頻道標出相關的事。

  3. 03

    它真的會動手：寫／合 PR、跑資料分析、處理 incident、bug triage、盯 metrics，不是聊天機器人。

  4. 04

    65% 數據：Anthropic 產品團隊現在 65% 的代碼來自內部版 Claude Tag（兩篇不同來源的報導一致）。

  5. 05

    權限按頻道隔離：法務群的分身看得到合約看不到研發代碼，越權直接拒絕；所有操作都記在它自己的帳號日誌。

一句話

這不是私人聊天機器人的進化，是把 Claude 變成組織級、有身分、有工具、能被信任動手的團隊成員。

Type AAI × Coding · Design × Code

Claude Design 2.0：設計與寫碼合流

商業本質（中文）
簡報 · In Brief核心 5 點
  1. 01

    帳單打通：Claude Design 與 Claude Code 額度全面整合，共用 5 小時會話＋週額度池；企業設計部與開發部不用各買一套，一套帳單就能算出投報率。

  2. 02

    一鍵匯出 PowerPoint：內建幻燈片工作流，產出帶講者備註、可全螢幕、多層級的檔案，一鍵匯出原生 PPT，直接對標微軟 Office；也能生 app 原型、線框圖、文件、動畫。

  3. 03

    頁內全功能編輯器：點畫布上任何元素就能改文字、字級、字體、粗斜；還有像 Figma 一樣的圖層視圖。

  4. 04

    新交互鐵律：模型負責 0→1 的框架構建，人類滑鼠負責 1→100 的微調，不用再為了一個像素的偏差反覆燒 token。

  5. 05

    Design Sync：在寫碼端設好品牌規範（字體／主色／風格），一個指令把整套「視覺基因」傳到設計端，之後每個按鈕、每個儀表板都忠實遵循品牌手冊。

一句話

關鍵不是它更會生圖，是把設計→代碼→活資料串成一個閉環，設計與開發的邊界被打掉了。
